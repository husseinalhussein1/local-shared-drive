بصفتك مهندس برمجيات Full-Stack خبير ومحترف في Next.js (App Router) وUI/UX Designer. أريدك أن تبني لي مشروعاً جامعياً متكاملاً وشغالاً بنسبة 100% (واجهات + باك إند + داتا بيز) باسم "Local Network Shared Drive".

⚠️ شرط برمي صارم وقاطع (مهم جداً): 
لا تستخدم الـ Server Actions أو أسلوب الـ MVC في ربط البيانات. يجب بناء الباك إند بالكامل على شكل RESTful API Endpoints منفصلة تماماً باستخدام (Next.js Route Handlers) داخل مجلد `app/api/`. والفرونت إند يقوم بالاتصال بها عبر طلبات (Fetch) ويرسل ويستقبل البيانات بصيغة JSON، لكي نتمكن من فحص الـ APIs واختبار الحماية والأكواد (مثل 401 و 403) عن طريق برنامج Postman بشكل مستقل تماماً أمام دكاترة الجامعة.

سياق التصميم المطلوب (مستوحى من Stitch):
- تصميم واجهات الفرونت إند كاملة داخل المشروع بناءً على مواصفات مجلد لدي اسمه `stitch_nexus_local_drive` وملف `design.md`.
- الثيم يجب أن يكون داكناً بالكامل ومودرن (Modern Tech Dark Mode): خلفية داكنة جداً (مثل `#0B0F19`)، بطاقات وعناصر بنظام الزجاج المضبب (Glassmorphism مع backdrop-blur)، وتأثيرات حركية ناعمة (Transitions). متجاوب بالكامل (Responsive) للابتوب والهاتف.

معمارية المجلدات وتخزين الملفات (الباك إند):
- قاعدة البيانات هي SQLite مع Prisma ORM.
- المجلدات هي "مجلدات وهمية" (Virtual Folders) تُدار وتُفصل داخل قاعدة البيانات فقط.
- يوجد مجلد فيزيائي واحد ثابت على الهارد باسم `uploads` في جذر المشروع، وكل الملفات المرفوعة تُحفظ بداخله فعلياً باستخدام حزمة `fs` الخاصة بـ Node.js، ويتم ربط كل ملف بالمجلد الوهمي التابع له عبر حقل `folderId` في قاعدة البيانات.

الأدوار والصلاحيات (3 مستويات):
1. `USER`: يرى فقط المجلدات المسموحة له ويحمل الملفات (Read-Only).
2. `USER_PLUS`: يستعرض، ويظهر له زر إنشاء مجلد وهمي، وزر رفع ملفات داخلها، وحذفها. عند إنشائه للمجلد، يظهر له خيار (Checkbox): "السماح لليوزر العادي برؤية المجلد".
3. `ADMIN`: يرى كل شيء دائماً، ويتحكم بصلاحيات أي مجلد، ويدخل لصفحة الإدارة ليرى الإحصائيات وسجل التحميلات. بالإضافة إلى (ميزة جديدة): يملك واجهة وفورم لإضافة مستخدمين جدد للنظام وتحديد أدوارهم (USER أو USER_PLUS).

المطلوب منك توليد الأكواد البرمجية الكاملة بدون اختصارات (Don't skip any code):

أولاً: خطوات التجهيز والأوامر (Setup Commands):
- الأوامر اللازمة لتثبيت Prisma ودمجها مع SQLite وتوليد الجداول وتثبيت مكتبات التشفير والـ JWT (مثل bcrypt و jose) وأيقونات Lucide React.

ثانياً: ملف إعداد قاعدة البيانات متوافق مع SQLite (`prisma/schema.prisma`):
- يضم جداول: User, Folder, File, DownloadLog.

ثالثاً: ملف الحماية المركزي (`middleware.js` في جذر المشروع):
- لحماية الصفحات والـ APIs وحظر الأدوار غير المصرح لها من دخول صفحة الإدارة `/admin` أو طلب الـ APIs الخاصة بها.

رابعاً: كود الـ APIs المنفصلة الكاملة (Next.js Route Handlers - REST API):
1. `POST /api/auth/login`: التحقق من الحساب، وتوليد JWT، وحفظه في `httpOnly Cookie`.
2. `POST /api/admin/users/create`: متاح فقط للـ ADMIN لإنشاء مستخدم جديد (USER أو USER_PLUS) وتشفير كلمت مروره قبل الحفظ.
3. `GET /api/folders`: جلب المجلدات والملفات المسموحة لكل مستخدم بناءً على دوره عبر طلب GET عادي يعيد JSON.
4. `POST /api/folders/create`: لإنشاء مجلد وهمي بالداتا بيز.
5. `POST /api/files/upload`: استقبال الملف وحفظه في مجلد `uploads` الفعلي وربطه بـ `folderId`.
6. `POST /api/files/download`: تسجيل الحركة في جدول `DownloadLog` ثم عمل Stream للملف لتحميله.
7. `GET /api/admin/logs`: جلب الإحصائيات وسجل التحميلات للآدمن.

خامساً: واجهات الفرونت إند الكاملة والمربوطة بالـ APIs عبر الـ Fetch (Pages):
1. صفحة تسجيل الدخول (`app/login/page.js`).
2. الصفحة الرئيسية للمستودع (`app/page.js`): تعرض المجلدات والملفات كـ Cards أنيقة مع إخفاء/إظهار الأزرار ديناميكياً حسب دور المستخدم.
3. لوحة تحكم الإدارة (`app/admin/page.js`): تعرض بطاقات الإحصائيات، جدول الحركات، وفورم/نافذة منبثقة (Modal) مخصصة للآدمن لإضافة مستخدمين جدد (يحدد الاسم، الإيميل، الباسورد، والدور).

سادساً: ملف الـ README.md (توثيق شامل، نار، وتفصيلي من البداية للنهاية باللغتين العربية والإنجليزية):
يجب أن يكون ملف الـ README مرجعاً كاملاً للدكتور ويحتوي بدقة على:
- مقدمة عن المشروع وأهدافه والتقنيات المستخدمة.
- شرح هندسة وتدفق البيانات (Data Flow) وكيفية تخزين الملفات في مجلد فيزيائي واحد `uploads` وفصلها وهمياً بالداتا بيز عبر الـ `folderId`.
- توثيق شامل لكل الـ APIs المذكورة أعلاه (الرابط، نوع الطلب، الصلاحية المطلوبة، الـ Body المرسل).
- تفصيل كامل وشامل للردود المتوقعة (Success Responses كود 200 أو 201 مع صيغة الـ JSON كاملة) وجميع ردود الأخطاء المتوقعة (401 Unauthorized عند غياب التوكن، 403 Forbidden عند محاولة مستخدم عادي طلب صلاحية آدمن، 404 Not Found، و 500 للمشاكل التقنية) مع كتابة الـ JSON الراجع لكل خطأ.
- دليل تشغيل المشروع على الشبكة المحلية (Local IP Configuration) لربط الهواتف واللابتوبات أثناء المناقشة.

المطلوب منك الآن:
توليد الأكواد والخطوات اللازمة لإضافة صفحة توثيق تفاعلية للمشروع عند المسار `app/api-docs/page.js` (أو مسار مناسب في App Router) لكي أتمكن أمام لجنة المناقشة من استعراض كافة الـ APIs وتجربتها حياً (Try it out) ورؤية الردود (JSON Responses).

يرجى توليد وإضافة التالي:

1. أوامر التثبيت (Installation Commands):
   - الأوامر اللازمة لتثبيت مكتبات التوثيق المناسبة لـ Next.js App Router (مثل swagger-ui-react أو المكتبة المودرن السريعة @scalar/nextjs المتوافقة مع الثيم الداكن).

2. ملف مواصفات الـ OpenAPI (Swagger Specification File):
   - كتابة ملف الإعداد أو الـ JSON/YAML الذي يوثق كافة الـ APIs التي أنشأناها في المشروع بدقة، وهي:
     * `POST /api/auth/login` (تسجيل الدخول وتوليد التوكن).
     * `POST /api/admin/users/create` (إنشاء مستخدم جديد - للآدمن فقط).
     * `GET /api/folders` (جلب المجلدات والملفات حسب الصلاحية).
     * `POST /api/folders/create` (إنشاء مجلد وهمي).
     * `POST /api/files/upload` (رفع ملف إلى مجلد uploads).
     * `POST /api/files/download` (تحميل ملف وتسجيله في الـ Log).
     * `GET /api/admin/logs` (جلب سجل الحركات والإحصائيات - للآدمن فقط).

3. كود واجهة التوثيق التفاعلية (`app/api-docs/page.js`):
   - كود الصفحة كاملة وثيمها متوافق مع المظهر الداكن والمودرن للمشروع (Dark Theme).
   - تأكد من تفعيل خاصية إرسال الـ Cookies (بما أننا نستخدم httpOnly Cookie للـ JWT) لكي تعمل تجربة الـ APIs (Try it out) بشكل حقيقي وصحيح عند تسجيل الدخول من داخل صفحة سواغر.

4. تحديث ملف الـ README.md:
   - إضافة قسم يشرح للدكتور كيفية الدخول لصفحة السوغر الحية وتجربة الـ APIs يدوياً عبر المتصفح مباشرة كإثبات على احترافية معمارية الباك إند (RESTful Architecture).

يرجى كتابة الأكواد كاملة وبدون اختصارات لتسهيل عملية النسخ واللصق الفورية.
